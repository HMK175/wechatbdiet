# 腾讯云 COS 部署（中国大陆访问）

GitHub Pages 在国内访问可能较慢或不稳定，可改用腾讯云对象存储（COS）的静态网站托管，国内访问更稳定。以下从创建存储桶开始。

**注意**：2024 年 1 月 1 日后创建的存储桶，不再支持通过 COS 默认域名（包括静态网站域名）直接访问对象，**必须绑定自定义域名**才能访问静态网站。部署前请准备好已备案的域名（中国大陆需备案）。

---

## 一、创建存储桶

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 进入 **对象存储 COS**：产品 → **对象存储**，或直接打开 [COS 控制台](https://console.cloud.tencent.com/cos5)
3. 点击 **创建存储桶**
4. 填写：
   - **名称**：例如 `fitness-diet`（需全局唯一，可加后缀如 `fitness-diet-1234567890`）
   - **所属地域**：选离你用户近的，如「华东」或「华南」
   - **访问权限**：选 **公有读私有写**（否则网站无法被访问）
   - 其他保持默认即可
5. 点击 **确定** 完成创建

---

## 二、开启静态网站托管

1. 在 **存储桶列表** 中点击刚创建的存储桶名称，进入详情
2. 左侧菜单找到 **基础配置** → **静态网站**
3. 点击 **编辑**，按下面勾选/填写后 **保存**

**需要勾选 / 填写的项：**

| 项 | 建议设置 | 说明 |
|----|----------|------|
| **当前状态** | **开启** | 保持打开（蓝色） |
| **索引文档** | `index.html` | 必填，作为首页 |
| **错误文档** | `index.html` | 单页应用建议填入口页，刷新或直接访问子路径时仍返回首页 |
| **错误文档响应码** | **200** | 选「200」而不是「原始错误码」，这样访问不存在的路径时返回 200 + index.html，由前端接管路由 |
| **强制 HTTPS** | **开启** | 建议打开，访问更安全 |
| **忽略 html 扩展名** | 可选 | 不需要可关；开启后访问 `index` 会匹配 `index.html` |

索引/错误文档的文件名、后缀大小写要和桶里实际文件一致（如 `index.html` 不要写成 `Index.HTML`）。

> **重要**：2024 年 1 月 1 日后创建的存储桶，**不再支持**通过 COS 默认域名（包括静态网站域名）直接访问对象。必须绑定 **自定义域名** 后才能访问静态网站，下面第四、五步会写如何绑定。

---

## 三、上传网站文件

需要上传的是「构建结果」里的静态文件，而不是整个 Git 仓库。

1. 在存储桶详情里，进入 **文件列表**
2. 确认根目录下要有：
   - `index.html`（入口页）
   - `manifest.json`
   - `service-worker.js`
   - 若有 `icon-192.png`、`icon-512.png` 等也一并上传
3. 上传方式二选一：
   - **控制台**：在文件列表点击「上传文件」，选择本地 `fitness diet` 目录下上述文件（注意：不要上传整个项目里的 `.git`、`node_modules` 等，只传页面用到的文件）
   - **COS 桌面工具 / CLI**：可用 [COSBrowser](https://cloud.tencent.com/document/product/436/11366) 或 [COSCMD](https://cloud.tencent.com/document/product/436/10976) 批量上传

上传后，根目录结构大致如下：

```
/
  index.html
  manifest.json
  service-worker.js
  (可选) icon-192.png, icon-512.png
```

---

## 四、绑定自定义域名（必做）

因腾讯云政策：**2024 年 1 月 1 日后创建的存储桶，不再支持通过 COS 默认域名（包括静态网站域名）直接访问对象**，所以必须绑定自定义域名才能访问你的静态网站。

1. 在存储桶详情，左侧进入 **域名与传输管理** → **自定义源站域名** 或 **静态网站** 相关域名配置
2. 点击 **添加域名** 或 **绑定域名**
3. 填写你要用的域名，例如：
   - `diet.yourdomain.com`（子域名）
   - 或你已备案的独立域名
4. **源站类型** 选择 **静态网站源站**（不要选默认的 COS 源站，否则索引文档不生效）
5. 若控制台有 **CDN 加速** 选项，建议开启，国内访问更快
6. 保存后，控制台会给出 **CNAME 记录值**（形如 `xxx.cos.ap-xxx.myqcloud.com` 或 CDN 的 CNAME）

---

## 五、在 DNS 里添加 CNAME

1. 到你购买域名的服务商（腾讯云 DNSPod、阿里云、Cloudflare 等）的 **DNS 解析** 里
2. 为该域名（或子域名）添加一条 **CNAME 记录**：
   - **主机记录**：若域名为 `diet.yourdomain.com`，主机记录填 `diet`；若用根域名则按服务商说明填写
   - **记录值**：填上一步控制台给的 CNAME 地址
3. 等待生效（几分钟到几十分钟不等）

---

## 六、HTTPS（推荐）

- 若使用腾讯云 **CDN** 绑定域名，一般在 CDN 控制台可申请 **免费 SSL 证书** 并开启 HTTPS
- 若仅用 COS 自定义源站域名，请在该域名/静态网站配置处查看是否支持 HTTPS 及证书配置

开启 HTTPS 后，用 `https://你的域名` 访问即可。

---

## 七、Supabase 回调地址（若用登录功能）

若站点使用 Supabase 邮箱登录，需把腾讯云最终访问地址加入 Redirect URLs：

1. 打开 [Supabase 项目](https://app.supabase.com) → **Authentication** → **URL Configuration**
2. 在 **Redirect URLs** 中新增：
   - `https://你的域名/`
   - `https://你的域名/index.html`
3. 保存

---

## 八、之后更新站点

代码更新后，重新上传覆盖 COS 中对应文件即可（尤其是 `index.html`、`service-worker.js`、`manifest.json`）。可继续用控制台上传，或 COSBrowser / COSCMD 脚本化上传。
